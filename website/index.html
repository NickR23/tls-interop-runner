TLS Interop Runner

About

When a new addition to TLS such as Delegated Credentials or Encrypted Client Hello is proposed, developers must not only implement it in their library, but also check that this implementation agrees in behaviour with those of other libraries. The TLS Interop Runner is a framework heavily inspired by the QUIC Interop Runner that automates this checking.

This page documents the current interop status as reported by the TLS Interop Runner.

Contributing

We welcome bugfixes, new features, and new implementations! The project can be found on GitHub. By adding a new implementation, you can make use of our infrastructure to test interop against the other implementations in the project. To do so, you can follow the examples set by existing implementations.
